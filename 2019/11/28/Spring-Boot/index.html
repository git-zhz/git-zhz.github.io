将一个大项目中的每一个功能提取出来，独立成为一个项目，每一个子项目之间通过网络协议进行连接，每一个子项目就是一个微服务。

springboot：用于快速开发微服务模块。

基于SpringBoot的Web应用

静态资源

springboot是一个jar，因此静态资源就不是放在webapp中，而是通过WebMvcAutoConfigration类中的addResiurceHandlers（）指定:/webjars

webjars把js、html、css等静态资源变成一个jar包

webjars官网：https://www.webjars.org/

以前引入静态资源是将这些资源下载放入到webapp目录中，springboot将这些静态资源直接以jar文件（Maven）的形式引入项目。

引入时：从jar目录结构的webjars开始写

如果自己写静态资源，如何放到springboot中：springboot约定将一些目录设置成静态资源目录，放入即可。

classpath：类路径：src/mian/resources

在ResourceProperties类中 { “classpath:/META-INF/resources/“,
“classpath:/resources/“, “classpath:/static/“, “classpath:/public/“ }

index.html 为默认首页

网站中：网页标签的logo是固定名字：favicon.ico 阅读源码可知只需要将favicon.ico文件放入任意静态资源文件中即可

（Propertites文件中的prefix+属性）自定义静态资源目录：在配置文件中：spring.resources.static-locations=classpath:/res/,classpath:/img/

以上就将静态资源的目录设置为了classpath:/res/,classpath:/img/，把原来约定的替换掉了

日志

常见的日志框架有：UCL、JUL、jboss-logging、logback、log4j、log4j2、slf4j

springboot默认使用：slf4j、logback，已经配置好，直接使用

指定日志显示格式：
a.日志显示在console中
logging.pattern.console=%d{yyyy-MM-dd} [%thread] %-5level %1ogger{5e} -%msg%n

%d：日期时间
%thread：线程名
%-5leve1：显示日志级别，-5表示从左显示5个字符宽度
%logger{50}：设置日志长度
%msg：日志消息
%n：回车
